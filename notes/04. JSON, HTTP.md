# JSON (JavaScript Object Notation)

### 优点

- 轻量；易读 (人和机器都)
- 无需编程语言环境
- 网络传输安全
- 支持嵌套 (arr in obj / obj in arr)
- 流动性读取，大规模数据也可以很快读取
- 跨平台 (web / mobile / IoT / databases)
- 可以转换成 js obj
	- `JSON.parse()` 把 JSON 变成 js obj
	- `JSON.stringify()` 把 js obj 变成 JSON
- 可延展：在原基础上添加新 field 即可，不会改变原结构


## 用途

- 代替数据库
- 作为在线 JSON API 供人使用
- 用 HTTP methods ( `GET, POST, PUT, DELETE` ) 来和 JSON-API 通讯
  来 manipulate fields (key: value)
	- regardless of which library / function / arroach ur using
	- e.g. Axios, jQuery, Node's http / https, fetch()

## JSON 格式注意
- 不可以是 class instance
- 不可以包含 function / undefined (可以是 null)
- 所有 key 必须加引号 quoted

# HTTP (Hypertext Transfer Protocol)

- HTTP is foundation of communication on the web
- 该协议定义了 how clients (e.g. browser) & servers exchange requests & responses to transfer data

## Request API
- 在浏览器里用 `window.fetch()` （直接写成 `fetch("api地址")` ）
- `fetch(api地址)` 返回一个 Promise
- 然后用 `await` e.g. `const dog = await fetch("dog.api地址");`
- 但 `await` 必须在 `async` 声明的函数内部才能生效
- `await` 后面是一个 Promise, 而不是真的值

### 方法2：Promise.then().catch()

**Promise.then().catch()用法（但不用写这个）**

```js
//////////////////////////////////////////////////////////////
// 定义 promise
const promise = new Promise((resolve, reject) => {

//🔴注意 resolve() 和 reject() 必须在不同条件下
//  如果直接写 resolve(); reject();，则第二行永远不会运行。
//  用 if-else 或者 try-catch

	try {
		const result = successExpression; // 在这里写要做的事情
		resolve(result);  // resolve() 里写的必须有值
	} catch (error) {
		reject(error);  // try 中出现的任何 error 都会传递给 catch
	} finally {
		// 可选，无论如何最后都会执行
		finalExpression;
	}
});

//////////////////////////////////////////////////////////////
// 后续 .then().catch()

promise.then(callback()).then(callback()).catch(callback());

promise
	// then1
	.then(
		// 这里 result1 叫什么都无所谓，都是从 resolve() 传出的值
		(result1) => {
			const result2 = expression(result1);
			return result2;
		}
	)
	
	// then2
	.then(
		// 这里 result2 叫什么都无所谓，都是从 then1 传出的值
		try {
			(result2) => expression(result2);
		} catch (error) {
			
			//🔴所有 error 都要 log, 一下为了给程序员看!!!
			console.error("Error in then2:" error);
			
			//🔴必须 throw, 不可 return error,
			//  只有 throw 才会把 ...then() 状态标记为 rejected
			//  后续 then 全都不再执行，直接最后 ...catch()
			//  但如果 return error, 当前状态仍为 fulfilled
			//  后续 then 会继续执行
			throw error;
		}
	)
	
	// then3
	.then(
		() => "也可以不使用传出的值"; // 直接显示文字
	)
	
	// catch
	.catch(
		// e: reject() 传出的值，或前面任何地方传出的 error
		(e) => console.error();
	);
```

### 方法3：Async / Await Keywords

```js
// async 声明的函数一定返回一个 promise
// define async function
async function fetchData() {
	
	// 暂停在这里，直到 fetch 完成
	// await API request
	const response = await fetch("https://example-api.com/");
		// fetch() 返回的也是 promise, 需要用 await 来解包他并后续操作
	
	// 拿到结果后继续执行
	// return json
	return response.json();
}

// call the async function, do then & catch
fetchData().then(()=>"success").catch((e)=>console.error("Error:",e));
```
