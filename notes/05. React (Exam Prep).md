## Virtual DOM (VDOM)
- Real DOM is the actual structure represented in the UI
- VDOM is the memory representation of DOM
- ==**Reconciliation:**== 
	- 直接改动 HTML -> DOM 会全部重新 render
	- compo 改动后 -> 生成新 VDOM -> 两版 VDOM 对比找到差异 -> DOM 只 re-render 差异部分

## Performance
- React Diffing Algorithm 把原本需要 $O(n^3)$ 的 time-complexity (时间复杂度) 缩减到了只有 $O(n)$ , 通过 hash-maps, key-value pairs
- denote key-value pairs using *Props*

## One-way Data Binding
- aka "single-inheritance"
- React 数据 / 属性只能从 Parent compo 传向 Child compo
- Parent compo 创造 made-up attributes (aka "Props") -> Child compo 接收

```jsx
// Parent.tsx
import Child from "./Child.tsx";  // 为了传值给 parent

export default function Parent() {
	let mood:string = "happy";
	return(
		<Child mood={mood} />  // 定义 prop 名为 mood, 传入 mood 值
	);
}
```


## Promise

### Declare & Define a Promise Obj

```js
// promise obj: 表示异步状态
// 表示一个现在不知道，未来才会有的值
const promise = new Promise(
	// executor function
	(resolve, reject) => {
	
		// Do something async here. e.g. third-party API call
		// fetch("api.com/data");
		
		if (greatSuccess) {
			resolve('success');  
			//🔴promise status turn to `fulfilled`
			//  and return 'success'
		} else {
			reject('failure');  
			//🔴promise status turn to `rejected`
			//  and return 'failure'
		};
	};
);
```

### Use The Promise Obj

```js
// 创建 promise 用来做一个可能会耗时的事情
const promise = new Promise(

	// promise executor function
	// resolve 和 reject 两个 func 是 promise executor func 的参数
	(resolve, reject) => {
	
		// 干事情
		这里做可能会延迟的事情，
		事情没干完之前 promise obj 都处于 pending 状态
		
	    // 成功/失败后的 message
	    if (在这写成功的条件) {
		    resolve('action succeeded, promise fulfilled');
		    // 执行 resolve() 后 promise 的状态变成 fulfilled
		    // resolve() 内的值会被传递给 .then() (下文中的 success)
	    
	    } else {
		    reject(new Error("action rejected"));
		    // 执行 reject() 后 promise 的状态变成 rejected
	    };
	};
);

// 处理 promise 返回的值
promise
	// 只有当 promise 的状态变成 fulfilled，才能触发.then() method
	.then(
		// success 是接收到的 resolve() 返回的值
		// 可以叫任何名字，比如 result / success
		success => {
			if (符合then1要干的事情的条件) {
				// 干 then1 的事情，对于 resolve 得到的
				// 结果(success)进行进一步操作
				return "then1: " + success
				
			} else { 
				// 如果不符合 then1 干事情的条件
				// 则在 then1 时报错
				throw new Error("then1 error");
			}
		}
	) //🔴注意千万不要加 ";" 。这里是链式 method 调用
	  //  相当于: promise.then().then().catch()
	
	.then(
		// 第二个 then, 接收第一个 then 的 return
		// 注意：必须有 return 才能接收到，如果只是 
		// console.log 的话接收不到，会是undefined
		result => {
			if (符合then2的条件) {
				console.log(result, "then2 executed")
			} else {
				throw new Error("then2 error")
			};
		};
	)
	
	.catch(
		// reject() 的结果，或者任何一个 thown error, 都会被 catch
		// 🔴catch() 必须放在最后，这样才能监听以上所有步骤
		error => {
			console.log("catch:", error || error.message);
		}
	)
```
