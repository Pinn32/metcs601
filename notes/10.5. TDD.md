# Test Driven Development

## TDD（测试驱动开发）

**一句话概括：先写测试，再写代码。**

正常开发顺序是：写代码 → 测试代码 TDD 的顺序是：**写测试 → 写代码 → 重构**

这个循环叫做 **红绿重构**：

- 🔴 **红**：写一个测试，运行，失败（因为功能还没写）
- 🟢 **绿**：写最少的代码让测试通过
- 🔵 **重构**：整理代码，保持测试通过

---

### 例子 1：写一个加法函数

**第一步：先写测试**

```python
def test_add():
    assert add(2, 3) == 5
```

运行 → ❌ 失败，因为 `add` 函数根本不存在

**第二步：写代码让测试通过**

```python
def add(a, b):
    return a + b
```

运行 → ✅ 通过

---

### 例子 2：写一个判断奇偶的函数

**先写测试：**

```python
def test_is_even():
    assert is_even(4) == True
    assert is_even(3) == False
```

运行 → ❌ 失败

**再写代码：**

```python
def is_even(n):
    return n % 2 == 0
```

运行 → ✅ 通过

---

### TDD 的好处

- **强迫你想清楚需求**，再动手写
- 代码天然有测试覆盖，**不怕改坏**
- 写出来的代码往往**更简洁**（只写够用的）
