### Canonical URLs 示例 (修改 Head)

> 多个 URL 如 `example.com/page?ref=twitter`、`example.com/page?utm=abc` ,
> 均可指向 `example.com/page`。

### ✅ 新版写法: App Router
```js
export const metadata = {
  title: "My Page",
  alternates: {
    canonical: "https://example.com/page",
  },
};
```
> - 自动生成: `<link rel="canonical" href="https://example.com/page" />`
> - 必须用 name export, 名称必须是 `metadata`
> - 可以放在任何地方, 比如 layout.tsx 里
> - ==注: metadata 只作用于当前文件夹及其子文件夹==, 若想根据page修改title, 在 page.tsx 里写 export const metadata

### ❌ 旧版写法: Pages Router
```jsx
import Head from "next/head";

export default function Page() {
  return (
    <>
      <Head>
        <link rel="canonical" href="https://example.com/page" />
      </Head>
      <div>Content</div>
    </>
  );
}
```

## 🔹 Routing Features

| Feature | 说明 |
|---------|------|
| 文件即路由 | 目录结构 = URL 结构，文件夹 + `page.tsx` 自动生成路由 |
| 动态路由参数 (dynamic) | 文件夹命名为 `[par-name]/`，支持动态 URL 参数 |
| 通配路由 (catch-all) | `[...slug]/` 同时处理多段 URL 或不同结构 |
| 路由遮蔽 (masking) | 可自定义区别于服务端文件结构的 URL 结构 |
| 预加载 (pre-fetch) | 自动预加载 linked pages，客户端无感知加载 |
| 浅路由 (shallow) | 修改 URL query 参数不触发数据重新获取 |
| API 路由 | 在应用内直接集成后端逻辑 |
| 自定义 404 | 创建自定义 404 页面 |

| 客户端 Auth 状态 | 用 `useEffect`/`useState` 管理客户端认证状态 |
| 第三方 Provider 集成 | 集成主流认证服务商 (auth providers) |

## 🔹 Root Layout

```tsx
// layout.tsx
import React from "react";

export default function RootLayout(
	{children, }:
		Readonly<{children: React.ReactNode;}>
){
	return (
		<html lang="en">
			<body>
				{children}
			</body>
		</html>
	)
}
```

参数 `{children, }` 末尾的逗号是 trailing comma，其他 prop 可能有 `params`（内含 `slug`）。

`: Readonly<T>` 是参数的 TS type，将 object 变成 immutable，意为"该组件接收一个含 `children` 的对象作为参数, 且这个对象参数不可 mutate"。

==注意：`Readonly<T>` 只能作为 object 的 type==，例如：
```tsx
Readonly<{
  prop1: type1;
  prop2: type2;
}>
```
