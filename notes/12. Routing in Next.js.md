# 🟣 Dynamic Routing
## 🔹Next.js

**放在文件夹里:** 
<code>next-app/app/.../<span class='red'>[spinal-case-par-name]</span>/page.tsx</code>

### ==useParams()==

`useParams()` 直接提取出当前路径中所有 `[par-name]`
可以用 `const {id} = useParams();` 来解构对象 (object destructuring)

```jsx
const params = useParams();

// fetch weather data with SWR
const {data, error} = useSWR(
	`/api/getWeatherData?city=${params.city}`,
	(url) => fetch(url).then((res) => res.json())
);
```

```jsx
const Root( = () => {

	)
}
```

# 🟣 Navigation
## 🔹 SSR (Server-Side Rendering)
```jsx
// 用一个 object 储存 dynamic URL props {name,href,icon}
const links = {
	{
		name: "Home",
		href: "/dashboard",
		icon: HomeIcon,
	},
	{
		name: "Invoices",
		href: "/dashboard/invoices",
		icon: DocumentDuplicateIcon,
	},
	{
		name: "Customers",
		href: "/dashboard/customers",
		icon: UserGroupIcon,
	},
};
```
## 🔹 CSR (Client-Side Rendering)
```jsx
// 用 usePathname() 来获取 pathname
// 用 .map() 来用 props 生成 links
export default function NavLinks() {
  const pathname = usePathname();

  return (
	<>
  	  {  // 👈🏻 此处{}代表嵌入js表达式(map)
	  	links.map((link) =>
		  <StyledDiv key={link.name}>
			<ThemeProvider theme={{isActive: pathname === link.href}}>
			  <StyledLink href={link.href}>
			    <StyledIcon as={link.icon} />
			    <span>{link.name}</span>
			  </StyledLink>
			</ThemeProvider>
		  </StyledDiv>
	  	)
	  }
	</>  {/* 👈🏻 最外面必须放上 fragment */}
  )
}
```

### ==usePathname()==
`usePathname()` 是 CSR hook，获取当前 URL 路径, 直接返回当前路径字符串, 比如 `'/dashboard/settings'`
> 注: 只返回路径, 不返回 query 参数 ( `?tab=1` ) 和 hash ( `#section` )

```jsx
'use client';  // 👈🏻 只能在 client-side 组件里用!!!
import { usePathname } from 'next/navigation';
const pathname = usePathname(); // 获取当前路径
```

### links.map()
`links.map((link) => XML)` 遍历 `links` 对象，为每条 link 渲染一个 `StyledDiv`。
直接写 XML 表达式, 无需用 `()` 或 `{}` 包裹, 若用 `{}` 必须写 `return()`

### StyledDiv: ==key== & ref

`key` 和 `ref` 都是 ==React 保留 prop==，所有组件均可添加，但不会被传递给子组件。

|  | `key` | `ref` |
|--|-------|-------|
| 给谁看 | React 内部 | 开发者自己 |
| 作用 | 识别元素身份，优化 DOM diff | 直接访问 DOM 元素 |
| 用法 | `key={link.name}` | `<input ref={inputRef}>` |
| 读取 | 不可读取 | `inputRef.current.value` |

### ==ThemeProvider==
`ThemeProvider` 是 ==styled-components 的一个组件==, 传入 `isActive: pathname === link.href`，当前路径匹配时 `isActive` 为 `true`，可在 styled-component 内用此值切换激活样式。
- `theme` 是 styled-components 预设的 prop 名, 通过 ThemeProvider 组件传给子组件
- `isActive` 是自定义的, 从 `<ThemeProvider ...>` 里传入: `theme={{子属性:值}}`

```jsx
// 导入
import styled from 'styled-components';
import { ThemeProvider } from 'styled-components';
```

```jsx
// StyledLink
const StyledLink = styled(Link)`
  color: ${({ theme }) => theme.isActive ? 'blue' : 'gray'};
`;
```

> `${}` 是 template literal, 用于在 CSS 中嵌入 JS 语法
> `({ theme }) => theme.isActive ? 'blue' : 'gray'` 意为 `theme` 这个 prop 的子prop `isActive` 为 `true` 时, 给 `color` 蓝色, 否则灰色

```jsx
// 通过 ThemeProvider 传 {color} 给 StyledLink
<ThemeProvider theme={{ isActive: pathname === link.href }}>
  <StyledLink href={link.href}>...</StyledLink>
</ThemeProvider>
```

> `{isActive: pathname === link.href}` => 选中当前页面的 Link, 改成高亮

### StyledLink: ==href==
注意这是一个 ==Next.js 的 Link 组件==, 不是 React 的 Link 组件
- `import Link from 'next/link'` 从 ==next/link== 导入 default
- `<Link href={...}> </Link>` 参数是 ==href==

### StyledIcon: ==as==

`as` 是 ==styled-components 的 prop==, 用来动态指定使用的资源组件

```jsx
const StyledIcon = styled.svg`...`;  // 假设默认是 svg
```

```jsx
<StyledIcon as={link.icon} />  // 实际渲染成 link.icon 那个组件
```

## 🔹 API Request Function

位置: `/app/api/getFooData/route.ts`

```tsx
export async function getFooData() {
  const apiKey = process.env.API_KEY;

  const res = await fetch("https://api.example.com/data", {
    headers: {
      Authorization: `Bearer ${apiKey}`,
    },
  });

  const data = await res.json();

  return Response.json(data);  // Response: Web API, HTTP Response
}
```

# 🟣 Special Files of Next.js
| 文件 | 作用 |
|------|------|
| `layout.tsx` | 当前 segment 及子页面的共享 UI |
| `page.tsx` | 当前路由的页面 UI，使路由公开可访问 |
| `loading.tsx` | 当前 segment 的加载状态 UI |
| `not-found.tsx` | 404 UI |
| `error.tsx` | 当前 segment 的错误 UI |
| `global-error.tsx` | 全局错误 UI |
| `route.ts` | 服务端 API endpoint |
| `template.tsx` | 每次导航重新渲染的布局 UI |
| `default.tsx` | 并行路由 (parallel routes) 的 fallback UI |

**[Doc: Routing Files](https://nextjs.org/docs/app/getting-started/project-structure#routing-files)**
