# Cookies
通过 HTTP Response 的 header (Set-Cookie) 来让客户端的浏览器保存缓存数据  
通常保存==登录状态==等身份识别信息, 让浏览器记住你

在 Next.js 服务器端写 cookie: 

```tsx
// @/app/api/.../routes.ts
import { NextResponse } from "next/server";

// 设置 GET method 让别人能够访问你网站的 API
export async function GET() {
	// 创建新 http response 实例
	const res = NextResponse.json({ message: "ok" });  // "ok" 可用在后端函数
	
	// 设置cookie
	res.cookies.set("token", "abc123", {
		httpOnly: true,    // 只可通讯, 不可被js读取
		secure: true,      // 只能通过 https (加密网络) 通讯
		sameSite: "lax",   // 跨站限制等级: 松
		path: "/",         // 此domain内, 全站可用
		maxAge: 60 * 60 * 24 * 7,  // 7天后过期 (persistent cookie)
	});
	
	return res;  // 返回带有 cookie 的 http 响应
}
```

# Security Cookies
## HTTP-Only Cookies (JS not allowed)
`httpOnly: true` 禁止 JS 读取 cookie, 比如运行 `document.cookie` 脚本时不可读取 cookie token

- state 由 server 生成, 储存在浏览器 cache 里
- **HTTP Cookie 的 CSRF (Cross-Site Request Forgery 跨站请求伪造) 预防措施:**
	- **增加严格度:** 增加 value assigned to `sameSite` from `lax` to `strict`  
		- **strict:** 所有点击链接跳转都不带 cookie
		- **lax:** 平衡安全 & 可用性 (默认用 lax)
		- **none:** 允许所有跨站请求带 cookie (必须配合 secure cookie)
	- **缩短期限:** 减少 shelf-life of the cookie, 通过给 `maxAge` 更低的数字来缩短 cookie 的期限

```tsx
res.cookies.set("oauth_state", state, {  // token -- "oauth_state": state
	httpOnly: true,
	secure: process.env.NODE_ENV === "producction",
	sameSite: "lax",  //🔴改成"strict"可以预防CSRF
	path: "/",
	maxAge: 10 * 60,  //🔴减小数字来缩短cookie寿命,预防攻击
});
```

## Secure Cookies
`secure: true` (只能按照 HTTPS 协议通信, ie. 只允许加密网络传输 cookie)

- 使用 Secure Cookie, 只能通过 HTTP==S== 连接, state 会重新送回, 并和本地 state 比较
```tsx
// 判断请求是否合法 (缺参数 / 参数不匹配)
if (!code || !returnedState || returnedState !== storedState) { 
	return NextResponse;  // 直接返回 HTTP 响应
}
```

|**条件**|**含义**|**说明**|
|---|---|---|
|`!code`|缺少 OAuth 授权码|登录回调没有 `code`，流程不完整|
|`!returnedState`|缺少 state|没有 state，无法做安全校验|
|`returnedState !== storedState`|state 不匹配|可能被篡改或 CSRF 攻击|


```tsx
const returnedState = url.searchParams.get("state");

const cookieStore = await cookies();
const storedState = coookieStore.get("oauth_state")?.value();

if (!code || !returnedState || returnedState !== storedState) {
	return NextResponse;
}
```

# Lifetime Cookies

## Session Cookies (会话 cookie)
- 储存在 RAM (cache) 内存缓存中; 只有浏览器打开时才储存, 关闭即清除
- 没有过期时限, 不写入磁盘, 关闭浏览器 = 自动清除缓存, 不残留状态
- RAM 比 SSD 读写快多了

```tsx
res.cookies.set("theme", "dark", {
	httpOnly: true,
	secure: process.env.NOTE_ENV === "production",
	sameSite: "lax",
	path: "/",
	maxAge: 60 * 60 * 24 * 7,  //=604800 seconds (7 days)
});
```

## Persistence Cookies
- 永久 cookie 写在 SSD 里 (hard drive 里), 即使关闭浏览器, 重启计算机, 也保存了

```tsx
res.cookies.set("carSession", "abc123", {
	httpOnly: true,
	secure: process.env.NOTE_ENV === "production",
	sameSite: "lax",
	path: "/",
	//🔴没有maxAge
});
```

## Cookie 征求用户同意
- Session Cookie 不必征求同意, 不被 GDPR (General Data Protection Regulation, legal compliance) 限制
- Persistence Cookie 必须要征求用户同意才能用
- (end-user: 终端用户)

# 其他 Cookies
## Zombie Cookies
僵尸cookie: 被删了还能复活的 cookie
- recreate themselves after deletion
- use multiple storage locations (RAM / SSD等等)
- Hard to remove

## Super Cookies
超级 cookie: 所有高级追踪技术的统称
- 用多个储存/缓存机制, 储存持久用户身份id, 即使删除 cookie 也继续追踪你
- 隐蔽, 分散储存

# Local Storage 本地存储

Local Storage: 浏览器本地==永久==保存键值对 (key:value pairs)
- 存储空间: 5-10Mb, 取决于浏览器
- 永久性: 永久保存, 即使关闭浏览器也存在
- 范围: 
	- 每个域名单独储存; 
	- 根据 HTTP/HTTPS 分别储存
- 几乎所有主流浏览器都支持本地储存

**Local Storage CRUD Methods:**  
`localStorage.methods(...)`

|**CRUD**|**Method**|
|---|---|
|Create / Update|`.setItem(key, value)`|
|Read|`.getItem(key)`|
|Delete|`.removeItem(key)`|
|Delete All|`.clear()`|

---

# Session Storage 会话储存

Session Storage: 浏览器本地==暂时==保存键值对 (key:value pairs), tab 关闭就删除
- 存储空间: 5Mb, 取决于浏览器
- 永久性: 暂时保存, 窗口关闭就删除
- 范围: 
	- 每个窗口 (tab) 单独储存
	- 不同窗口之间不共享!
- 所有主流浏览器都支持会话储存

**Session Storage CRUD Methods:**  
和 `localStorage` 一样, 改成 `sessionStorage`

|**CRUD**|**Method**|
|---|---|
|Create / Update|`.setItem(key, value)`|
|Read|`.getItem(key)`|
|Delete|`.removeItem(key)`|
|Delete All|`.clear()`|

# Cookies / Session Storage / Local Storage 对比

|**特性**|**Cookies**|**Session Storage**|**Local Storage**|
|---|---|---|---|
|生命周期|可设成会话or永久|关页失效|永久, 只能手动删除|
|是否发给服务器|是|否|否|
|JS可访问|有时|可|可|
|服务端可访问|是|否|否|
|存储大小|~4KB|~5MB|5~10MB|
|安全性|中 (可设置安全flags)|一般|一般|

- Local Storage: 长期浏览器数据
- Session Storage: 暂时页面数据
- Cookies: 服务器需要记忆用户端数据

# CORS (Cross-Origin Resource Sharing)
- CORS: 让不同 origin 之间也能互相访问资源
- **Origin:** 
	- protocol (协议: http/https) 
	- domain (域名)
	- port (端口)  
	  (e.g. `localhost:3000` 和 `localhost:5000` 就是不同 port)

```tsx
export async function OPTIONS() {
	return new Response(null, {
		status: 204,
		headers: {
			"Access-Control-Allow-Origin": "*",  //任何origin都能访问
			"Access-Control-Allow-Methods": "GET",
			"Access-Control-Allow-Headers": "*",
		},
	})
}
```
